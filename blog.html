Important FAQ

What is the Document Object Model (DOM)?

The DOM (Document Object Model) is a programming interface for web documents. It represents the structure of a webpage as a tree of objects, where each object corresponds to part of the document (like elements, attributes, and text). This allows programming languages like JavaScript to manipulate the content, structure, and style of a webpage dynamically.

How do you select an element from the DOM?

1.document.getElementById('myId'): Selects by ID.
2.document.querySelector('.myClass'): Selects the first matching element by CSS selector.
3.document.getElementsByClassName('myClass'): Selects elements by class name.
4.document.getElementsByTagName('div'): Selects elements by tag name.

What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique in the DOM where a single event listener is added to a parent element instead of individual listeners on each child element. When an event occurs, it bubbles up from the target child to the parent, allowing the parent to handle the event.

Benefits:
1.Efficiency: Reduces memory usage by minimizing the number of event listeners.
2.Dynamic Elements: Automatically handles events for newly added child elements without needing to reattach listeners.
3.Simplified Code: Makes it easier to manage events for groups of elements.

How do you manipulate an element's attributes and styles using the DOM?

1.Set Attributes
element.setAttribute('attributeName', 'value');
2.Get Attributes
const value = element.getAttribute('attributeName');
3.Common Attributes (Direct Properties)
element.id = 'newId'; // Set ID
element.className = 'newClass'; // Set class
4.Styles
element.style.propertyName = 'value'; // Example
element.style.backgroundColor = 'blue';
element.style.fontSize = '16px';
